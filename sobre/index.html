Quem sou

Eu sou o André — engenheiro de software em transição de carreira, construindo base técnica sólida e projetos reais. Em 2024, aos 41 anos, eu decidi recomeçar na tecnologia de forma séria: não como hobby, mas como um caminho profissional.

Minha trajetória anterior me deu um repertório grande em comunicação, estratégia, organização e execução. Só que, na prática, o que me trouxe para a programação foi o mesmo impulso de sempre: entender como as coisas funcionam, estruturar sistemas e transformar ideias em algo que roda, resolve problema e pode evoluir.

Hoje atuo como desenvolvedor web full stack, criando sites, sistemas e aplicações web que funcionam de ponta a ponta. Gosto de entender o problema antes do código e transformar ideias em soluções simples, organizadas e sustentáveis.

Foco técnico hoje

Backend (minha base principal)

Java + Spring Boot, APIs REST, JPA/Hibernate, PostgreSQL, validações, DTOs, organização em camadas (Controller/Service/Repository) e boas práticas de modelagem.

Frontend (UI limpa e consistente)

HTML, CSS e JavaScript com mentalidade mobile-first, e evolução para interfaces em React quando faz sentido. Gosto de componentes simples, tipografia bem cuidada e layout que “respira”.

Testes, qualidade e evolução

Começando pelos fundamentos: testes unitários (JUnit/Mockito), organização de código, legibilidade, e documentação enxuta para permitir manutenção e crescimento do projeto.

Como eu trabalho

Mobile-first de verdade

Eu começo pelo essencial no mobile (conteúdo, hierarquia, espaçamento), e só depois amplio para desktop. Isso evita “remendos” e mantém o layout consistente.

Arquitetura simples, mas intencional

Prefiro uma estrutura clara do que complexidade cedo demais. Camadas bem definidas, nomes consistentes e responsabilidades explícitas.

Documentação que serve

Registro decisões técnicas, padrão de pastas, setup do projeto e pontos importantes. Documentar bem ajuda o projeto a continuar “fácil” mesmo quando cresce.
